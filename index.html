Token Management
Proactive Auto-Refresh
Tokens are refreshed 120 seconds before expiry. No tool call ever hits a live 401. Thread-safe TOKEN_STATE with threading.Lock ensures concurrent safety.
Transport
Streamable HTTP / SSE
Uses MCP's Streamable HTTP transport at /mcp. A separate /health/token-status endpoint lets clients verify token validity before starting a session.
OAuth 2.0
Full OAuth Flow Built-in
Server exposes /auth/url and /auth/exchange HTTP endpoints. Client opens browser, pastes redirect URL — tokens are stored server-side automatically.
Data Scopes
4 Google Health Scopes
Activity & fitness, health metrics & measurements, sleep, and nutrition — requested with access_type=offline for long-lived refresh tokens.
Aggregation
Daily Rollups & Custom Ranges
Every data type supports dailyRollUp (civil-day aggregates) and rollUp (arbitrary time ranges) in addition to raw data point listing.
Extensibility
Generic Passthrough Tools
list_data_points, daily_rollup, and rollup tools accept any data type string — future-proof against new Google Health endpoints.

Token Lifecycle

✓ Fresh
expiry > 120s away
Token is returned immediately with zero API calls. Fast path in _token() — the common case for most tool calls.
⚠ Buffer Window
0s < expiry ≤ 120s
Proactive refresh triggered before expiry hits. The tool call that prompted this never fails — refresh happens inline before returning the token.
✗ Expired
expiry ≤ 0s
Refresh grant executed using stored refresh_token. If no refresh token exists, raises with clear OAuth instructions. Full re-auth only needed once.
server.py — _token() logic 
def _token() -> str:
    secs = _seconds_until_expiry()

    if secs is not None and secs > REFRESH_BUFFER_SECS:
        # ✓ Fresh — fast path, no API call needed
        return TOKEN_STATE["access_token"]

    if secs is not None and secs <= 0:
        _do_refresh()           # Fully expired → must refresh
    elif secs is None:
        if has_refresh_token():
            _do_refresh()       # Unknown expiry + refresh token → refresh
    else:
        _do_refresh()           # Within 120s buffer → proactive refresh

    return TOKEN_STATE["access_token"]
03

OAuth Flow

01
Client checks token validity
GET /health/token-status → { valid: bool, expires_in: int, has_refresh_token: bool }
02
If invalid → get authorization URL
GET /auth/url → { url: "https://accounts.google.com/o/oauth2/v2/auth?..." }
03
Browser opens, user consents
webbrowser.open(url) — user approves 4 health scopes in Google's consent screen
04
User pastes redirect URL
Full redirect URL (or just the code= value) pasted into the terminal prompt
05
Server exchanges code for tokens
POST /auth/exchange → extracts code, calls Google Token URL, stores access + refresh tokens in TOKEN_STATE
06
All tool calls now work
MCP session starts — agent calls health tools transparently. Auto-refresh runs as needed.
client.py — run_oauth_flow() 
def run_oauth_flow():
    # 1. Get the auth URL from the server
    resp = requests.get(f"{BASE}/auth/url", timeout=5)
    url  = resp.json()["url"]

    # 2. Open browser for user consent
    webbrowser.open(url)
    redirect_url = input("Paste the full redirect URL here: ")

    # 3. Exchange code for tokens (server handles extraction)
    requests.post(
        f"{BASE}/auth/exchange",
        json={"authorization_code": redirect_url},
    )
    # Tokens stored in TOKEN_STATE — all tools work now ✓

Setup & Installation

1. Clone the repo and install dependencies

terminal 
# Install dependencies
pip install mcp requests crewai \
  crewai-tools langchain-groq \
  python-dotenv uvicorn starlette

# Or use requirements.txt
pip install -r requirements.txt

2. Configure environment variables

.env 
GOOGLE_HEALTH_CLIENT_ID=your-client-id
GOOGLE_HEALTH_CLIENT_SECRET=your-secret
GOOGLE_HEALTH_REDIRECT_URI=http://localhost

AZURE_API_KEY=your-azure-key
AZURE_API_BASE=https://your-endpoint.openai.azure.com
AZURE_API_VERSION=2023-05-15

3. Start the MCP server

terminal — server 
python server.py
# → Starting on http://127.0.0.1:8000
# → MCP endpoint: http://127.0.0.1:8000/mcp
# → Token status: GET /health/token-status

4. Run the CrewAI client

terminal — client 
python client.py
# If no token: browser opens automatically
# Paste redirect URL → tokens stored
# Chat starts immediately

You: Show my step counts for this week
You: What was my average resting heart rate in May?
You: Summarise my sleep from last 7 days

Google Cloud Console setup

1. Create a project at console.cloud.google.com
2. Enable Google Health API (health.googleapis.com)
3. Create OAuth 2.0 credentials
4. Add https://www.google.com to Authorized Redirect URIs
5. Add test users under OAuth consent screen if app is in testing mode
